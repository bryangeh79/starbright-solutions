# 04_risks_issues.md

## 当前已知风险

- 技术栈尚未定稿，后续工程结构需要先锁定
- 数据模型尚未细化到字段级别
- 第三方渠道接入依赖平台授权、Webhook 能力与账号条件
- 权限模型存在后期膨胀风险，必须保持中小企业可维护范围
- 知识库范围有失控风险，必须限制在 FAQ、产品资料、服务资料、公司介绍与文件问答内

## 必须在 Phase 2 内解决的问题

- 选定前台、后台、API 与数据库方案
- 明确主要数据对象及关联关系
- 确认渠道接入的统一抽象方式
- 定义基础权限边界与团队协作结构
- 定义知识库、线索、转人工与通知之间的数据流

## 风险控制原则

- 不做复杂微服务拆分
- 不做 Enterprise 级权限体系
- 不提前扩展到电商、支付或 ERP 场景
- 先保证可交付、可维护、可扩展，再考虑增强功能
