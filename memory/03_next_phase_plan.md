# 03_next_phase_plan.md

## 下一阶段

- Phase 2｜技术架构与项目骨架

## Phase 2 目标

- 确定可落地、可维护、可扩展的中小企业级技术方案
- 将 Phase 1 的蓝图转化为工程结构
- 形成后续开发可以直接开工的目录、数据模型与接口规划

## Phase 2 执行顺序

1. 锁定技术栈
2. 细化数据对象与关系
3. 规划 API / service / module 划分
4. 定义环境配置与部署方式
5. 搭建前后端项目骨架
6. 为后续 Phase 3 的对话与知识库 MVP 预留扩展位

## 下一步唯一优先动作

- 进入技术栈选择与项目骨架规划

## 为什么先做技术栈 / 数据结构 / API 规划

- 技术栈决定后续代码组织方式、部署方式与维护成本
- 数据结构决定会话、客户、线索、知识库、通知和报表是否能顺畅联通
- API 规划决定前台、后台、渠道接入和对话引擎能否统一协作
- 这三项若不先定，后续实现会反复返工，影响 Phase 3 及之后的开发效率
